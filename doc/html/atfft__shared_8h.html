typedef double atfft_sample

A typedef defining the type used for real values in ATFFT.

By default this will be double, but by providing additional definitions at compile time this can be changed.

Definition Sample Type
-D ATFFT_TYPE_FLOAT float
-D ATFFT_TYPE_LONG_DOUBLE long double
-D ATFFT_TYPE_DOUBLE double

These definitions take precedence in the order given in the above table, i.e. if ATFFT_TYPE_FLOAT is defined, atfft_sample will always be typedefed to float, even if the other two definitions are present.

In order for atfft_sample to be double, either none of the above definitions must be present or only ATFFT_TYPE_DOUBLE must be defined.

typedef atfft_complex_d atfft_complex

A typedef defining the type used for complex values in ATFFT.

Complex values are stored as an array of two values (a real and an imaginary part). Typedefs providing a complex type for each precision of floating point value are described above (atfft_complex_f, atfft_complex_d, atfft_complex_l). atfft_complex is typedefed to one of those three, such that its real and imaginary part have the same type as atfft_sample.

By default atfft_complex will be an array of two doubles (atfft_complex_d), but by providing additional definitions at compile time this can be changed.

Definition Complex Type
-D ATFFT_TYPE_FLOAT atfft_complex_f
-D ATFFT_TYPE_LONG_DOUBLE atfft_complex_l
-D ATFFT_TYPE_DOUBLE atfft_complex_d

These definitions take precedence in the order given in the above table, i.e. if ATFFT_TYPE_FLOAT is defined, atfft_complex will always be typedefed to atfft_complex_f, even if the other two definitions are present.

In order for atfft_complex to be atfft_complex_d, either none of the above definitions must be present or only ATFFT_TYPE_DOUBLE must be defined.
